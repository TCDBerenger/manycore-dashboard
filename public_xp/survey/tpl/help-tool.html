About Visualisation

You will now be presented with a visualisation tool that shows performance data along with the source-code of the programs. Please read the following to understand how the visualisation work and what is shown on the screen.

Proportional View

This view allows you to compare various components as a proportional estimation of CPU activity, adding up to 100%. The pane on the left allows to show/hide unnecessary dimensions and applies the selection to every view on the screen.

Timeline View

This view allows you to visualise the programs running over time. The color-scheme is exactly the same as in the proportional view showed above.

Worker View

This view allows you to see the same data per thread. Each line represents a particular thread and each line can be colored by:

  • Locality: a function that represents the effectiveness of data-locality patterns, variying from green to red. Green represents an optimal data locality (ie: low cache and tlb misses and no page faults) while red represents the opposite.
  • Process: each thread is colored by the process it belongs to.
  • Thread: each thread is given a unique color.

In addition to the previous dimensions that represent CPU Activity (eg: Loading from L2) there are "Thread", "Process" and "Core Time". The first two axes show the thread ID and the process it belongs to while the "Core Time" shows a percentage of time the thread has spent on a core, executing code or waiting for data.
