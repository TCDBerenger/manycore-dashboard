Locality of reference, also known as the principle of locality, is a phenomenon describing the same value, or related storage locations, being frequently accessed. There are two basic types of reference locality â€“ temporal and spatial locality. Temporal locality refers to the reuse of specific data, and/or resources, within a relatively small time duration. Spatial locality refers to the use of data elements within relatively close storage locations. Sequential locality, a special case of spatial locality, occurs when data elements are arranged and accessed linearly, such as, traversing the elements in a one-dimensional array.

Consider a hypothetical computer with a fast, single-line cache of 8 bytes (analogous to a CPU cache) and a slow buffer (analogous to RAM) in which a 4x4 greyscale image data resides to illustrate the example. This simplified cache system can only cache data in blocks of 8 bytes (an 8-byte cache line).

First iteration of the loop touches 4 bytes (#0, #4, #8 and #12), two cache misses occur including the first load, as the image data needs to be loaded from memory into the cache line for CPU to use it. During the entire execution of the program, the total of 8 cache misses occurs. This means that our hypothetical CPU loads 8 * 8 = 64 bytes in total from our slow buffer into the cache.

Based on our taxonomy, we build a tool to visualise and compare (two) profiles.

A profile is a set of values harvested while a program/software/application running. The data comes from OS and harware.

 

Dashboard: overview

The top-level of our webapp is the Dashboard where you could compare the profiles and, more important, have a overview of all categories (first level in our taxonomy). The aim is to quickly identify one or more categories involved in the issue(s).

Once catogories are identified, you could drill-down to find the real cause of the issue.

 

Details of a category: drill-down

In each page, there is a set of graph that could help you to find the root of the problem.

To help you, you can customise each graph via the "Customise" tab in order to adapt and manipulate the data shown.

Also you coul find some interesting information in the "help" tab in particular if you are not familiar with the concept shown.

You're now going to try the tool wit two samples. Each one is a comparison of the same program with a sequential version and an parallel (optimised) version.

The first sample is "merge and sort" and the second is "particules".

Take your time, but it's only an help to understand our tool. So the real experiment starts after this samples.
