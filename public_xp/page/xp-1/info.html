In this experiment, you will be presented with data on the performance of some parallel programs. This performance data is obtained from hardware counters and visualised in a number of different ways. There are lots of different types of performance problems in parallel programs, and so these visualisations are based around a previously developed taxonomy of parallel performance problems. Take a look through the taxonomy and sub-problems below to familiarise yourself before beginning the experiment.

The tool can be used to visualize data from a single program, or to compare data from two different versions of a program. We refer to a data set for a particular program version as a profile.

 

Dashboard: overview

The top-level of the tool provides a Dashboard where you can compare profiles and have an overview of all categories (the first level in our taxonomy). The aim is to quickly identify one or more categories in which the problem may lie.

Once a category is identified, you can drill-down to investigate the potential cause of the issue.

 

Category: drill-down into problems

For each category of our taxonomy, a dedicated page displays a set of graphs that can help you to find the root of the problem.

You can customise many of the graphs via the "Customise" tab in order to change the way the data is shown.

More information is available in the "help" tab if you are having trouble interpreting any particular visualization.
