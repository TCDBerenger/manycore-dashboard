In this experimentation, you will test our approach and tool about the problem-centric software performance analysis on many-core (multicore) systems.

 

Parallel performance

In software engineering, there is several way to get clues about the efficiency and accuracy of a particular software. Here, we're focusing on the performances by analysing several indicators and measurments of a run of an application. Differents runs could produce differents values mostly because there are context-depending. But the analysis should be quite similar.

Our approach is problem-focused instead of hardware-focued. Based on surveys, we create a taxonomy for this kind of problems (see below). It is composed by 6 categories grouping about 4 problems.

[XP.tool.1] Based on our taxonomy, we build a tool to visualise the profile of a program run; it also allows to compare two profiles.

 

Dashboard: overview

[XP.tool.3] The top-level of our webapp is the Dashboard where you could compare the profiles and, more important, have a overview of all categories (first level in our taxonomy). The aim is to quickly identify one or more categories involved in the issue(s).

[XP.tool.4] Once catogories are identified, you could drill-down to find the real cause of the issue.

 

Category: drill-down into problems

[XP.tool.5] For each category of our taxonomy, a dedicated page displays a set of graph that could help you to find the root of the problem.

[XP.tool.6] To help you, you can customise each graph via the "Customise" tab in order to adapt and manipulate the data shown.

[XP.tool.7] Also you coul find some interesting information in the "help" tab in particular if you are not familiar with the concept shown.
